The Referral Code System
General Description:
The system shall assign a unique, permanent Referral Code to every registered user. This code can be shared and entered by other users during the enrollment process. When a code is used a specific number of times (T=3) for events, the owner of the code (the Ambassador) receives a Free Enrollment Credit.
1. Navigation and Interface
•	1: The system shall display the user's unique Ambassador Code in their Profile section.
•	2: The system shall provide a "Referral History" dashboard showing who used the code and for which events.
•	3: The enrollment form shall include an optional text field for entering a "Referral Code."
2. Referral & Reward Logic (Technical Complexity)
•	4: The system shall generate a unique alphanumeric code for each user upon account creation (e.g., USER123).
•	5: When a "Referral Code" is entered during enrollment, the system shall validate:
o	5.1: The code exists and belongs to a valid user.
o	5.2: The user entering the code is not the owner of the code.
•	6: Each successful use shall be logged as a ReferralInteraction linked to a specific EventID.
•	7: Once the system detects 3 successful usages of a code for a specific event, the Ambassador is granted a Free Enrollment for that specific event (or a future one).
