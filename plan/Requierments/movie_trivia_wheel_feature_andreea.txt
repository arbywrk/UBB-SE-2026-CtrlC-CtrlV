MW. Movie Trivia Wheel Feature

General Description

Movie Trivia Wheel is a gamification feature designed to increase user engagement on the platform by allowing users to spin a wheel and receive a set of movie-related trivia questions.

When the user spins the wheel, a trivia category is randomly selected. The user will then receive a sequence of trivia questions related to that category. Questions are multiple-choice and test the user’s knowledge of movies, actors, directors, awards, and film history.

Users who successfully answer all questions correctly will receive a reward in the form of a free ticket for a movie event available on the platform.

The feature also includes a hint system that allows users to eliminate incorrect answers during difficult questions.

1. Navigation and Interface

MW.1
The home page must contain a Movie Trivia Wheel section that includes a short description of the feature and a Spin the Wheel button.

MW.2
Pressing the Spin button opens the Trivia Wheel interface, which displays a wheel containing multiple trivia categories.

MW.3
The wheel must contain at least the following categories:

Actors
Directors
Movie Quotes
Oscars and Awards
General Movie Trivia

MW.4
The interface must include:

- the wheel visualization
- a Spin button
- the number of remaining spins available for the user
- a short explanation of the reward rules

MW.5
Pressing the Spin button triggers a wheel spinning animation, after which one category is randomly selected and displayed to the user.

MW.6
After the category is selected, the interface transitions to the Trivia Question screen.

2. Trivia Question Flow

MW.7
Each trivia session consists of 20 multiple-choice questions belonging to the category selected by the wheel.

MW.8
Questions are displayed one at a time and must include:

- the question text
- four possible answers
- exactly one correct answer.

MW.9
Users must select one answer before moving to the next question.

MW.10
Immediately after the answer is submitted, the system evaluates it and updates the current score counter displayed on the screen.

MW.11
Questions cannot be skipped and cannot be revisited after submission.

MW.12
The trivia session ends when all questions have been answered or when the user exits the session.

MW.13
At the end of the session, a results screen displays:

- total number of correct answers
- whether the user has earned a reward

3. Trivia Question Selection Logic

MW.14
Trivia questions are stored in the database together with the following attributes:

- question text
- category
- four answer options
- correct answer identifier.

MW.15
When a trivia session starts, the application randomly selects 20 questions belonging to the category chosen by the wheel.

MW.16
The selection process must ensure that the same question cannot appear more than once within the same trivia session.

MW.17
The order of the selected questions must also be randomized before they are presented to the user.

4. Hint System

MW.18
Each trivia session provides the user with one hint token.

MW.19
The hint can be activated during any question using a Hint button displayed near the answer options.

MW.20
When the hint is activated, the application must remove two incorrect answers from the available options.

MW.21
After the hint is used, the Hint button becomes disabled for the remainder of the session.

MW.22
The interface must clearly indicate whether the hint has already been used.

5. Reward Logic

MW.23
A user receives a free movie event ticket reward only if all 20 trivia questions are answered correctly.

MW.24
When a reward is generated, it must be stored in the user’s account together with:

- reward type
- redemption status.

MW.25
The reward can be applied during checkout when the user selects a movie event.

MW.26
After the reward is used, the system must mark it as redeemed to prevent reuse.

MW.27
A reward generated by the trivia wheel can only be used once.

6. Spin Limitations

MW.28
Each user is allowed one trivia wheel spin per day.

MW.29
The application must store the timestamp of the user’s most recent spin.

MW.30
If the user attempts to spin the wheel again before the next daily reset, the spin action must be blocked.

MW.31
In this case the interface must display a message informing the user when the next spin will become available.

MW.32
The daily spin counter resets automatically at 00:00 server time.
