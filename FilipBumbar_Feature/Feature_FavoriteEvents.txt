FE. Favorite Events & Notification System

General Description:
The system shall provide a "Favorite Events" feature that allows users to easily keep track of events they are interested in. Users can mark events as favorites and access them later from a dedicated page. The system will also monitor important changes related to those events (such as price decreases or new available seats) and notify the user. The purpose of this feature is to improve the user experience by allowing users to follow specific events without repeatedly searching for them in the application.

1. Navigation and Interface

FE.1: The system shall provide a "Favorite" button on the page of every event. This button allows the user to quickly add the event to their personal list of favorite events.

FE.2: When the user selects the Favorite button, the system shall store the selected event in the user's favorites list in the database.

FE.3: The system shall provide a dedicated "My Favorites" page where the user can view all events they have marked as favorites. The page shall display the basic event information such as title, date, location and price.

FE.4: The system shall allow the user to remove an event from the favorites list by pressing a "Remove from Favorites" button.


2. Favorite Management Logic

FE.5: The system shall ensure that the same event cannot be added twice to the favorites list for the same user. If the user tries to add the same event again, the system shall ignore the request.

FE.6: The system shall store favorite events separately for each user. This ensures that every user has their own independent list of favorite events.

FE.7: The system shall only allow events that exist in the database to be added to the favorites list.

FE.8: The system shall allow the user to view all favorite events at any time through the "My Favorites" page.


3. Notification Logic

FE.9: The system shall monitor changes related to the events saved in the user's favorites list.

FE.10: The system shall notify the user when the price of a favorite event decreases.

FE.11: The system shall notify the user when new seats become available for an event that was previously full.

FE.12: Notifications shall be displayed inside the application interface so that the user can quickly react and join the event if they are interested.


4. Example Business Logic

This feature involves several simple but useful business rules. The system must verify that the same event is not added multiple times to a user's favorites list. The system must also check for changes in the event data (such as price updates or seat availability) and generate a notification when such changes occur. These rules ensure that the feature behaves consistently and provides meaningful updates to the user.
